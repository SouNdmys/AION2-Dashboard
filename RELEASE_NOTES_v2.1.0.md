# AION2 Dashboard v2.1.0

发布日期：2026-02-27

## 本次版本定位

- `v2.1.0` 为“架构重构收口 + 功能热修”联合发布版。
- 保持业务规则口径稳定，在不再继续结构性拆分的前提下，完成关键体验与联动修复。

## 架构重构收口（已合入）

- renderer 端完成模块化收口：
  - 视图组件、交互 handlers、派生模型 hooks 分层清晰化。
  - 右侧面板与工坊子面板拆分后，跨卡片联动逻辑复用度提升。
- main/workshop-store 端完成 domain 化拆分：
  - `catalog / ocr / pricing / simulation / store` 领域职责独立。
  - 价格历史、信号、异常、快照变更链路从 core 中解耦，便于后续热修小范围落点。
- IPC 合约统一：
  - preload 调用桥接改为共享 contract 生成，错误格式与 payload 校验统一。
- 质量基线：
  - unit 与 smoke 回归链路纳入发布流程执行。

## 功能热修

- 圣域开箱奥德消耗调整：
  - `40 -> 80`。
- 做装模拟器材料联动增强：
  - 点击材料后同步更新市场分析器与右侧“历史价格管理”焦点项。
  - 交互不触发页面跳转或上拉下拉滚动。
- 做装模拟器取价冲突修复：
  - 修复手动改价后被 OCR/快照价格覆盖的问题，手动价格可稳定生效。
- 做装模拟器按钮合并：
  - “运行模拟”整合保存价格/库存并重算语义，减少重复操作入口。
- 角色星标：
  - 左侧角色列表新增 `★` 星标能力。
  - 优先级待办对星标角色增加排序加权，但仍保留原有任务权重/进度优先逻辑。
- 历史价格管理：
  - 新增“一键清空全部历史价格”。
- 版本与更新体验：
  - 工具栏新增版本号常驻显示与“检查更新”入口。
  - 手动检查发现新版本后，下载完成将直接重启安装。
  - 修正 Release 资产命名与 `latest.yml` 一致性，恢复自动更新链路可用性。

- 已执行 `npm run test:unit`
