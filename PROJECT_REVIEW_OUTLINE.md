## 8. 最关键的问题与踩坑

### 8.1 技术坑

- OCR / ONNX 安装版路径问题
- 自动更新 `latest.yml` 与安装包哈希不一致
- UI 重构过程中旧样式残留
- 状态逻辑和手动录入逻辑冲突

### 8.2 流程坑

- 在未冻结结构前功能容易变形
- 没有固定发布校验时容易出元数据问题
- UI 如果不按 phase 推进，容易越改越乱

## 9. 做对了什么

- 每个热修尽量小 commit
- 每次改完都跑 `typecheck / test:unit`
- 发布前固定跑 `test:smoke`
- 重要阶段都留文档与日志
- UI 改造按 phase 推进而不是一把梭
- 先收信息架构，再谈美化

## 10. 如果重来一次会怎么做

- 最早期就先冻结命名和边界
- 更早把发布脚本与哈希校验补上
- 更早约束 UI 只围绕主任务展开
- 更早建立“哪些不做”的清单
