# UI Phase 15 任务计划

日期：2026-03-09

## 目标

- 继续压缩“角色操作”页的纵向高度，让主视觉回到任务管理本身。
- 为“角色概览总览”恢复低饱和语义色，不再只靠红字提示高优先级。
- 保持浅色极简基底，不回退到大面积重色块。

## 执行顺序

### 1. 角色总览卡语义强化

- 为 `高优先 / 副本 / 周常 / 使命 / 休闲` 建立稳定的浅色语义底色。
- 任务行增加轻语义底和左侧强调条。
- 每张角色卡头部增加单一重点提示，不堆叠多个 badge。
- 保留任务细节列表，不再压成纯摘要卡。

### 2. 角色操作页顶部减重

- 把“当前角色 + 周统计 + 角色资料 + 快捷操作”重排为更紧凑的工具带结构。
- 让“返回角色总览”“保存资料”“同步回廊”等动作进入同一操作层。
- 保留信息完整度，但减少卡片数量和标题重复。

### 3. 任务区再压一轮密度

- 任务卡改成更扁平的“标题 / 一行摘要 / 状态 / 操作”结构。
- 分类区块上下留白继续收紧。
- 圣域保持单独识别，但和副本任务的节奏统一。

### 4. 资源区收口

- 保留当前 `奥德能量 + 微风商店 + 变换记录` 的关联布局。
- 继续减少说明文案和无效空白，让资源录入更像工具条。

### 5. 回归验证

- `npm run typecheck`
- `npm run build`
- `npm run test:smoke`

## 验收重点

- 角色页顶部是否明显变矮，首屏更快进入任务区。
- 总览卡是否恢复“看一眼就知道重点”的视觉识别。
- 高优先项是否不再只靠红字，而是靠轻语义块面来突出。
- 总体是否仍然维持当前浅色、干净、不过度装饰的方向。
