# AION2 Dashboard UI 改造执行单 v2（行为导向 + 极简）

版本基线：`v2.1.3`  
目标风格：`ChatGPT Web/macOS` 的低噪音、强层级、渐进披露。

## 1. 目标

1. 把“做装决策”变成单线流程：`选配方 -> 刷价格 -> 运行模拟 -> 决策`。
2. 不改业务逻辑与 IPC/store，只改 renderer 的信息架构和视觉层。
3. 将工坊默认体验从“参数面板集合”升级为“任务流界面”。

## 2. 范围与约束

1. 仅改 `src/renderer/src/**` 展示层与交互层。
2. 不改：
   - `src/main/**` 规则与算法
   - `src/shared/**` 业务类型与计算口径
3. 每个阶段必须独立 commit，并通过：
   - `npm run typecheck`
   - `npm run test:unit`

## 3. 关键问题（当前）

1. 主任务不聚焦：OCR/分析/库存高级配置长期占据首屏。
2. 渐进披露缺失：专业参数默认展开，认知负担高。
3. 市场分析卡过重：筛选、曲线、信号、触发区同时展示。
4. 跨卡片上下文分散：物品/分类状态在多处输入，来回跳。
5. 视觉系统不统一：按钮、层级、强调色使用密度偏高。

## 4. 目标信息架构（工坊）

1. 默认模式：`快捷模式`
   - 卡片 1：做装模拟（主流程）
   - 卡片 2：价格输入（轻量 OCR + 最近导入结果）
   - 卡片 3：结论摘要（利润、风险、建议动作）
2. 高级模式：`专业模式`（折叠区）
   - OCR 校准
   - 信号规则细项
   - 逆向推荐与批量管理
3. 右侧历史管理保持联动，不新增页面跳转。

## 5. 执行阶段

## Phase 0：基线冻结

1. 固化现状截图（桌面 + 窄屏）：
   - Dashboard 三页
   - Workshop 四卡片
2. 记录首屏主操作路径与平均点击步数。
3. 产出“改造前基线说明”。

## Phase 1：Design Tokens 收敛

1. 统一 `styles.css` 变量：
   - 颜色、间距、圆角、阴影、控件高度
2. 统一按钮与输入控件规格：
   - `.task-btn` 主操作
   - `.pill-btn` 次操作
3. 降低发光与高饱和提示密度。

验收：
1. 同类型控件高度/圆角一致。
2. 视觉噪音明显下降。

## Phase 2：工坊 IA 重排（不动逻辑）

1. 工坊主区改为两层：
   - 默认显示快捷模式
   - 专业模式可折叠展开
2. 模拟器卡只保留一条主操作路径：
   - `运行模拟（含保存）`
3. OCR 面板默认折叠高级区：
   - 热键、自动抓价保留
   - 校准与调参进入“高级设置”

验收：
1. 首屏可在 10 秒内完成一次“选配方 -> 运行模拟”。
2. 不需要滚动即可找到主按钮。

## Phase 3：市场分析减负

1. 默认仅保留：
   - 当前物品最近价格
   - 趋势方向
   - 异常提醒
2. 曲线与信号列表放入“展开查看”。
3. 统一卡片头结构：标题 + 一句话说明 + 右侧主/次操作。

验收：
1. 市场分析首屏高度下降。
2. 用户不读长文案也能完成操作。

## Phase 4：结果页优先

1. 模拟结果区首屏只留 4 指标：
   - 材料成本
   - 税后收入
   - 净利润
   - 利润率
2. 明细材料表折叠并按需展开。
3. 风险提示（缺价/可疑）统一样式与位置。

验收：
1. 首屏“是否值得做”一眼可判定。
2. 明细不再抢占主视觉。

## Phase 5：全局一致性收尾

1. Dashboard / Workshop 顶栏与卡片规范统一。
2. 焦点态与键盘可访问性补齐（focus-visible）。
3. 动效收敛到 `120~180ms`，避免漂移感。

验收：
1. 全站按钮与输入风格一致。
2. 操作节奏稳定，无突兀动画。

## 6. 每阶段交付模板

1. 改动摘要（3~6 行）
2. 文件清单
3. 设计取舍说明（为什么）
4. 回归结果：
   - `npm run typecheck`
   - `npm run test:unit`
5. 前后对比截图（桌面 + 窄屏）

## 7. 执行顺序（固定）

1. Phase 0
2. Phase 1
3. Phase 2
4. Phase 3
5. Phase 4
6. Phase 5

说明：禁止跳阶段并行改，避免风格反复与回归成本扩大。
