# AION2 UI Reference Strategy

## 目标

这份文档不是要“照抄大厂 UI”，而是给 `AION2 Dashboard` 定一套稳定的参考来源和学习边界。

当前产品类型：

- 桌面工作台
- 高频查看 + 高频录入
- 多角色任务管理
- 做装模拟 / 市场辅助

所以参考重点应该是：

- 信息架构
- 页面节奏
- 状态层级
- 工具化外壳

而不是营销感、品牌大片感或重装饰动效。

## 参考组合

### 1. Linear

负责：

- 主界面骨架
- 信息密度
- 任务列表层级
- 侧栏与内容区关系
- 状态优先级表达

适合迁移到本项目的部分：

- 左侧导航的目录感
- 总览卡的任务分组结构
- 列表型信息的压缩排版
- 主次按钮层级

不建议照搬的部分：

- 深色主题的默认氛围
- 过强的工程工具感文案

### 2. Notion

负责：

- 浅色极简底盘
- 留白
- 中性配色
- 低干扰容器
- 长时间使用的舒适感

适合迁移到本项目的部分：

- 页面背景与卡片关系
- 容器的轻边框和浅阴影
- 字号与行高节奏
- 输入框和表单的克制感

不建议照搬的部分：

- 过于“内容编辑器”式的空旷布局
- 太弱的任务型强调

### 3. Raycast

负责：

- 桌面客户端气质
- 侧栏收起/展开
- 工具条
- 按钮点击感
- 快捷操作区的效率感

适合迁移到本项目的部分：

- 侧栏像工具壳，而不只是堆按钮
- 紧凑型操作条
- 弹窗与次级工具入口
- 图标按钮和目录式导航

不建议照搬的部分：

- 过暗的主界面底色
- 过强的命令面板感

## 参考权重

推荐权重：

- `Linear`: 60%
- `Notion`: 25%
- `Raycast`: 15%

解释：

- 骨架像 `Linear`
- 气质像 `Notion`
- 桌面细节像 `Raycast`

## 设计原则

### 1. 主任务永远先于工具

用户打开应用，第一眼要先看到：

- 今天/这周做什么
- 这个角色现在还有什么能做
- 这件装备值不值得做

而不是先看到：

- 大量筛选器
- 大量设置项
- 大量辅助工具

### 2. 页面要像工作台，不像后台系统

目标体验：

- 打开就能进入当前工作
- 导航明确
- 内容区安静
- 状态重点清楚

避免：

- 大面积卡片堆叠
- 所有功能同层展示
- 强营销式视觉

### 3. 重点靠块面，不靠红字

当前产品最适合的重点表达方式：

- 淡语义背景
- 左侧细强调条
- 状态 pill
- 清晰的分组标题

不适合：

- 大面积高饱和红
- 只改数字颜色
- 所有项目都高亮

### 4. 工具感要来自结构，而不是装饰

“大厂感”通常不是更多特效，而是：

- 对齐好
- 层级清楚
- 颜色克制
- 间距稳定
- 交互节奏一致

## 页面映射

### 左侧栏

目标参考：

- `Linear` 导航结构
- `Raycast` 桌面工具壳

应具备：

- 更清楚的 app identity
- 导航和管理分区
- 折叠后仍保留强目录感

### 角色总览

目标参考：

- `Linear` 列表型看板
- `Notion` 轻量卡片底盘

应具备：

- 快速录入像工具条，而不是大表单
- 角色卡像任务摘要卡，而不是表格拼盘
- 分组清楚，但不过度花

### 角色操作

目标参考：

- `Linear` 信息分层
- `Raycast` 紧凑操作条

应具备：

- 顶部工具区扁平化
- 主任务卡节奏统一
- 资源区是“状态 + 录入”，不是堆说明

### 做装模拟

目标参考：

- `Notion` 主内容舒适感
- `Linear` 单任务页面结构

应具备：

- 首屏只保留主任务
- 市场工具后撤一层
- 结果区按真实操作顺序排列

## 绝对不学的方向

### 1. 官网化

不做：

- 大 banner
- 大插画
- 强品牌渐变
- 宣传页排版

原因：这不是官网，是工具。

### 2. 监控面板化

不做：

- 太多统计卡
- 太多图表并列
- 太多高饱和状态色

原因：你这个产品的核心不是“监控”，而是“决策 + 执行”。

### 3. 过度拟物或玻璃化

不做：

- 重玻璃
- 夸张阴影
- 复杂发光
- 过强景深

原因：容易显廉价，而且长时间使用会累。

## 可执行清单

### 第一轮

- 收敛全局中性色和阴影
- 强化左侧栏导航壳
- 压缩总览页快速录入和筛选条

### 第二轮

- 统一角色页工具条和任务卡节奏
- 再收一次市场工具内部样式

### 第三轮

- 让侧栏、总览、角色页、做装模拟使用同一套“工作台”语言
- 用更少的说明文案表达更多的状态信息
- 保持主任务优先，所有专业工具都后撤一层

## 本项目的最终落点

如果这套参考组合用对了，`AION2 Dashboard` 最终应该给人这几个印象：

- 像一个成熟的桌面工作台，而不是网页后台
- 看起来安静，但重点明确
- 信息很多，但扫一眼就知道先做什么
- 录入动作顺手，不需要到处找入口

判断是否做到位，可以用这四条自测：

- 左侧栏是否像目录，而不是按钮堆
- 中间内容是否先出现主任务，而不是设置项
- 高优先信息是否靠块面和层级突出，而不是单纯红字
- 页面切换后是否仍然像同一个产品，而不是几套风格混在一起
- 统一表格、输入框、按钮的细节语言

### 第三轮

- 做一套真正完整的 `light / dark` 双主题
- 补整套截图基线
- 固化为下一项目可复用的 token 规范

## 最终判断标准

不是“像不像某个大厂”，而是：

1. 打开页面时，主任务是否足够清楚
2. 重点信息是否能一眼扫到
3. 页面是否安静耐看
4. 录入和查看是否都顺手
5. 用两小时后会不会累

如果这五条成立，格调自然会上去。
